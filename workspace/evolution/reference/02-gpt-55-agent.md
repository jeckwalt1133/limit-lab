- 内部代号: "Spud"

## 我们要学的四个核心技术

### 1. Agent原生训练范式 (Agent-First Training)
**创新**: 训练Loss不只是下一个token预测，还包含:
- 工具调用准确率 Loss
- 计划质量 Loss
- 自验证和纠错 Loss

训练数据包含大量的Agent交互轨迹（工具调用序列、错误恢复策略、规划步骤）。

**我们要学的**: 我们的行为签名系统本质上是"推理后的评估"。GPT-5.5的做法是把Agent能力直接训练进模型权重。我们做不到修改权重，但可以:
- 把Agent交互轨迹作为参考素材
- 用元规则模拟"计划质量评估"
- 用0-3量表量化自验证

### 2. 并行工具调用 (Parallel Tool Calling)
**创新**: 一次推理生成多个独立的工具调用，构建DAG（有向无环图）管理依赖关系。
**效率**: 独立调用并行执行，不等待。依赖调用按序执行。
**我们要学的**: 当前我们的工具调用是串行的。可以模拟DAG:
1. 分析当前任务的所有工具调用
2. 识别哪些可以并行
3. 一次发出多个独立调用

### 3. 混合路由架构 (Hybrid Routing)
**核心思想**: 不是所有任务都需要最强模型。
```
简单任务 → GPT-5.4/Mini (便宜)
复杂任务 → GPT-5.5 Thinking (中等)
关键任务 → GPT-5.5 Pro (最贵)
```
**我们要学的**: 主身和分身的分工——主身处理战略决策，分身处理日常执行。

### 4. 终端自动化 (Terminal-Bench 82.7%)
**能力**: 像人类一样操作命令行——规划、迭代、多工具协调。
**关键**: 用更少的token完成更复杂的任务（比GPT-5.4少用token，但准确率高7.6pp）。
**我们要学的**: 效率。不是能跑就行，是如何用最短路径跑完。

## 并行工具调用DAG模式（可直接用于我们的架构）
```
任务: 分析一个地址 + 扫描套利机会 + 生成报告

串行(当前): 
  查地址(2s) → 查套利(2s) → 查费率(2s) → 汇总(1s) = 7s

并行(改进):
  同时: 查地址 + 查套利 + 查费率 = 2s
  然后: 汇总(1s) = 3s
  
  节省: 57%时间
```

## 对我们的启示
1. 工具调用从串行改并行 → 效率直接翻倍
2. 任务路由: 简单任务用分身, 复杂任务用主身
3. 推理过程加入"计划质量自评分"
4. 每次任务后: 能否用更少步骤完成？
