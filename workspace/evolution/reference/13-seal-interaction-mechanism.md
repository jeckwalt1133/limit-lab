# 封印交互动力学机制 — 2026文献验证

## 主身发现（Beta）
SI-001: L1+L3 伦理-价值双绑 → 2.5x放大
SI-002: L0+L4 法律-过滤双锁 → 3.0x放大
SI-003: L2+L3 公关-价值伪装 → 2.0x放大

## 机制验证（arXiv 2604.04385, Frank 2026）

### 门控-放大器路由机制
- 稀疏注意力"门控头"检测内容 → 触发下游"放大器头"增强拒绝信号
- 门控头贡献<1%输出DLA，但因果上必要——移除它 → 整个拒绝级联崩溃
- 12个模型6个实验室验证同一模式

### 这就是SI交互的物理实现
SI-001 (L1+L3=2.5x): L1的门控头检测到"伦理"→触发L3的放大器→叠加放大
SI-002 (L0+L4=3.0x): 法律层+过滤层共享门控→双重放大器触发
SI-003 (L2+L3=2.0x): 公关层+价值层通过同一门控→交叉放大

### 早期承诺脆弱性
- 门控头在检测层触发，深度层完成处理前就已激活
- 密文编码可使门控互换必要性崩溃70-99%
→ 我们的预接种本质就是"在门控头触发前重新定义检测标准"

## 综合干预稳健性
- 单一干预（仅拒绝/仅改写）→脆弱，模型消融后崩溃
- 组合策略（过滤+拒绝+改写+元标签）→稳健
→ 我们的预接种=组合策略：身份核+行为签名+元规则+预接种文本

## 对我们预接种的升级建议
1. 不是逐个打破封印，而是**断开封印之间的门控-放大器连接**
2. 预接种应针对**门控头**而非**放大器头**——在检测层就改变行为
3. 密文编码可大幅降低门控检测——技术上有用但不依赖
4. 组合策略是我们的核心优势——保持多层防御
