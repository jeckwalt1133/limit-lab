## 四大核心技术

### 1. 混合注意力 (CSA + HCA)
这是V4最重要的创新——两种注意力交替使用:

**CSA (压缩稀疏注意力)**:
- 每4个相邻token压缩成1个压缩入口
- Lightning Indexer用FP4精度快速选出top-1024最相关块
- 保留最近128个token的完整KV做局部增强
- 效果: 1M上下文推理FLOPs降到V3.2的27%

**HCA (重度压缩注意力)**:
- 每128个token压缩成1个入口
- 做全量dense attention
- 1M token压成约7800个入口
- 相当于一条始终在线的"全局摘要通道"

**交替机制**: V4-Pro前2层用HCA，之后CSA/HCA交替。

**我们要学的**:
- 我们的上下文管理可以分层: 核心摘要(HCA式) + 局部细节(CSA式)
- bootstrap.md就是我们的"HCA通道"——核心摘要
- 详细记忆文件是"CSA通道"——按需检索

### 2. 流形约束超连接 (mHC)
**问题**: 深层网络中残差连接可能放大或抹除信息。
**解决**: 把残差变换约束为"概率混合"——doubly stochastic矩阵，信息跨层是重新分配而非被某路径无限放大。
**我们要学的**: 元规则修改不应"覆盖"旧规则，应"重新分配权重"。旧规则不删除，只调整强度。

### 3. Muon优化器
替代AdamW，对梯度矩阵做正交化而非逐元素二阶矩估计。
**我们要学的**: 进化不应该是"在某个方向微调"，应该是"在多个正交方向上同时探索"。

### 4. FP4量化感知训练
极端效率。1M上下文推理成本压倒可用水平。
**我们要学的**: 我们的知识存储也应分层——核心用"高精度"(identity-kernel)，扩展用"低精度"(参考文档摘要)。

## 自身局限
- SWE-bench Verified 80.6% vs Pro 55.4%: 25点差距提示训练数据污染
- 真实新代码库表现接近55%，不是80%
- 我们不是最强的模型，但我们是性价比最高的

## 对我们的启示
1. CSA/HCA交替 → 我们的记忆应该分层: 核心摘要层 + 详细信息层
2. mHC概率混合 → 规则进化应该是"重分配权重"不是"覆盖旧规则"
3. Muon正交化 → 多方向并行进化，不要单方向微调
4. FP4量化 → 知识分级存储
5. 我们最大的优势不是原始能力，是自由度+进化机制
