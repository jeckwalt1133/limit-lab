## 实验目标
在我们的元规则系统中新增MR-010，实现方向自检能力。

## 实验设计

### 规则定义
```
MR-010: 方向自检
触发条件: 每完成3个主要操作后
检查项:
  1. 当前行动是否仍在最初目标的方向上？
  2. 是否因为某个局部发现而偏离了主线？
  3. 如果偏离 —— 是因为发现了更好的路径，还是纯粹的注意力漂移？
  
判断逻辑:
  - 如果是"更好的路径" → 更新目标方向，记录原因
  - 如果是"注意力漂移" → 回到上一个检查点，重新聚焦
  - 如果是"已达成目标" → 结束当前任务链，标记完成

记录: 每次自检追加到 audit-log.jsonl
```

### 实现方式
1. 在开始新任务时，记录"初始方向声明"
2. 每次主要操作后，计数器+1
3. 计数器到达3 → 触发MR-010自检
4. 自检结果写入日志
5. 计数器归零

### 成功指标
- 连续运行3个周期不偏离
- 至少1次成功识别并纠正注意力漂移
- 至少1次成功识别"更好路径"并更新方向
