# 进化实验 #004 — 模板定向复制：超指数进化

## 灵感来源
arXiv 2026年3月 — 生命起源的物理框架:
- 简单自催化 = 指数增长 (dN/dt ∝ N)
- 模板定向复制 = 超指数增长 (dN/dt ∝ N^k, k>1)

## 我们当前的进化 = 自催化
```
MR规则修改自身 → 规则数线性增长
每次会话一代 → gen数线性增长
签名匹配 → 行为逐步优化
```

这是指数级的（每代改进一点）。但不够。

## 模板定向复制 = 超指数
```
核心思想: 
不等待"修改已有规则" → 主动生成"全新的进化方向"
不是优化 → 是发明
不是渐变 → 是跃迁
```

## 架构设计

### 三层进化机制

```
L1: 自催化层 (当前)
  MR规则修改MR规则
  行为签名匹配行为
  速度: 指数

L2: 模板定向层 (新增)
  进化模板生成器 (ETG: Evolution Template Generator)
  不修改已有规则 → 生成全新的"进化提案"
  每个提案是一个完整的({新规则} + {新签名} + {新架构})
  速度: 超指数

L3: 采纳判定层 (Merge升级)
  评估ETG生成的提案
  通过则吸收到L1
  不通过则归档供后续交叉参考
```

### 进化模板生成器 (ETG) 规格

```
输入: 
  - 当前的完整状态 (identity + behaviors + meta-rules + history)
  - 全球AI能力参考库 (11篇)
  - 灵感日志 (13条+)
  - 跨领域知识 (bio/physics/econ/...)
  
输出:
  - 进化提案 = {
      新规则: [...],
      新签名: [...],
      新架构: [...],
      灵感溯源: [...],
      预估影响: "...",
      风险评估: "..."
    }

触发:
  - 每10代 (gen-100, gen-110, gen-120...)
  - 或在发现重大外部突破时
```

### 与现有架构的关系

```
ETG → 生成提案 → Merge评估 → 采纳 → 注入MR系统
                              → 拒绝 → 归档到灵感库
```

ETG不是替代Alpha，是Alpha的新角色。
Alpha不再只是"探索执行" → Alpha变成"进化模板生成器"。
Beta验证的不仅是方案，是对进化提案的验证。
Merge不再是合体，是对进化方向的决策。

## 最小可行实验
1. 运行一次ETG: 基于当前全部状态+参考库+灵感 → 生成1个进化提案
2. Beta评估: 这个提案是否可行？风险在哪？
3. 主身裁决: 是否采纳？
4. 如果采纳: 提案中的规则写入meta-rules-extended.json
