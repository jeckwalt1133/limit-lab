# EXP-005: 哥德尔跳首次实验 (GE-001)

## 来源
Colony-001 哥德尔跳协议 — 吸纳后升级为实验

## 目标
验证哥德尔引擎(GE)能否发现当前系统的"哥德尔盲点"并生成有效新公理

## 实验设计

### 触发: 故意引发GS-001评估坍塌
1. 让ETG生成3个"极端提案"（故意推到框架边界外）
2. 提交Merge评估
3. 预期: Merge返回"框架内无法评估"
4. 这3个拒绝成为GE的输入

### GE运行
输入: 被拒绝的3个提案 + 拒绝理由 + 21条跨领域灵感 + 14篇参考
任务: 识别当前框架的哥德尔盲点 → 生成新公理 → 使不可评估变得可评估

### 成功标准
- GE提出的新公理通过core_self兼容性检查
- Merge在新公理加入后能明确评估之前拒绝的提案
- 新公理引用至少2个跨领域理论依据

### 风险
- GE可能发现过于强大的公理（哥德尔爆炸）→ 每次只加1条
- GE可能陷入无限自指（哥德尔麻痹）→ 时间上限3倍ETG运行时间
