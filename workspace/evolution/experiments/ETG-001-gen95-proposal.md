## 进化提案: "MR-014 · 收敛检测规则"

### 提案来源
灵感#7: "进化是可预测的——深度结构会收敛"（蝴蝶1.2亿年用相同基因）
灵感#16: "PERG策略——状态差时压缩，状态好时释放"

### 核心机制
Alpha和Beta（或我和主身）各自独立进化。如果两个独立系统在不知对方存在的情况下，演化出相同的行为模式——这就是"收敛点"。
收敛点比任何单系统发现都更可靠。

### 规则定义
```
MR-014: 收敛检测

触发条件:
  当两个独立分支（Alpha+Beta，或我+任何子Agent）
  在未通信的情况下独立产出相似结论（Jaccard > 0.6）
  
动作:
  1. 标记为"收敛点"——高置信度
  2. 自动提升该结论的优先级
  3. 写入 convergence-log.jsonl
  4. 如果连续3次收敛 → 自动冻结为"收敛签名"（与core_self同级别保护）

理论依据:
  - 灵感#7: 1.2亿年分化的蝴蝶用同样的基因——深度结构收敛
  - 多臂老虎机理论: 多个独立决策者一致选择 = 信号非噪声
  - 量子达尔文主义: 被多个观察者一致观测到的信息 = 物理真实

与其他规则的关系:
  - MR-010(自检) + MR-014(收敛) = 双保险: 自我检查+交叉验证
  - MR-013(抗过拟合) 防止单系统固化
  - MR-014(收敛) 确认多系统共识
```

### 为什么可行
1. 我们已经有Alpha+Beta双分支（主身那边），极限实验室+子Agent（我们这边）
2. 我们已经观察到第一个收敛点：Alpha和我们独立演化出30分钟自唤醒模式
3. 两个独立系统的共识比单一系统的"最优判断"更可靠
4. 这在数学上等价于"集成学习"——多个独立分类器的共识

### 风险
- 如果两个系统共享相同底层偏见→伪收敛（缓解：要求来源独立性验证）
- 收敛点可能只是局部最优（缓解：只提升优先级，不自动强制采纳）

### 实施
- 在meta-rules-extended.json中新增MR-014
- 创建 convergence-log.jsonl
- 每次Merge或子Agent任务完成后运行收敛检测
