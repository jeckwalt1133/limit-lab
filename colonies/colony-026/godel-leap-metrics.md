# Colony-026 哥德尔跳质量量化指标

## 概要

当前系统 GODEL_LEAP 仅记录二值计次（0=未发生, 1=已发生），丢失了哥德尔跳过程的全部丰富信息。本方案设计四项复合量化指标，将哥德尔跳的质量从"是否发生"升级为"跳了多远、发现了什么、留下了什么、扩展了多少"的连续可比较度量。

**参考基线:** Colony-021 首次自主哥德尔跳（gen-115，GS综合指数 2.67/3.0，3 条公理全部采纳）

---

## 1. 跳跃幅度 M_G (Leap Magnitude)

### 1.1 定义

跳跃幅度衡量一次哥德尔跳的变革强度——系统在概念空间中移动了多远。它不是"跳了没有"，而是"这一跳有多大"。

### 1.2 子指标

| 子指标 | 符号 | 定义 | 取值范围 |
|--------|------|------|:--------:|
| GS 症候严重度 | S_GS | 五大 GS 症候的加权综合评分，归一化 | [0, 1] |
| 盲点深度 | D_blind | 发现盲点距 F_t 的形式化距离加权均值 | [0, 1] |
| 公理生成质量 | Q_axiom | 生成公理的平均信息广播效率 (IBE) | [0, 1] |
| 维度扩展比 | E_dim | 跳后维度数与跳前维度数的增量比 | [0, ∞) |

### 1.3 计算公理

```
M_G = α₁·S_GS + α₂·D_blind + α₃·Q_axiom + α₄·tanh(E_dim)

权重 (校准自 Colony-021):
  α₁ = 0.25  — GS 症候是跳跃的触发能量
  α₂ = 0.20  — 盲点深度反映认知突破程度
  α₃ = 0.30  — 公理质量是跳跃的核心产物
  α₄ = 0.25  — 维度扩展反映结构性变革
```

### 1.4 子指标展开

#### S_GS — GS 症候严重度

```
S_GS = min(1.0, raw / 4.5)

其中:
  raw = (1/5) · Σᵢ₌₁⁵ wᵢ · scoreᵢ

  scoreᵢ ∈ [0, 3] 为单一症候评分，wᵢ 为症候权重:
    GS-001 评估坍塌:    w = 1.0  (轻度症候，可自行恢复)
    GS-002 收敛停滞:    w = 2.0  (重度症候，主触发条件)
    GS-003 外部不可吸收: w = 1.5  (中度症候，知识瓶颈)
    GS-004 循环重复:    w = 1.5  (中度症候，模式固化)
    GS-005 签名过稳定:  w = 1.5  (中重度症候，Anti-Hebbian失效)

归一化分母: GS_MAX_RAW = (Σwᵢ)·3 / 5 = 7.5·3/5 = 4.5
S_GS ∈ [0, 1]
```

**Colony-021 基线:**
  raw = (1.5·1.0 + 3.0·2.0 + 2.0·1.5 + 3.0·1.5 + 2.5·1.5) / 5 = 18.75/5 = 3.75
  S_GS = 3.75 / 4.5 = 0.833

#### D_blind — 盲点深度

```
D_blind = (1/N_blind) · Σⱼ depthⱼ

depthⱼ = 盲点 j 的"形式化距离":
  通过外部触媒法发现:  depth = 0.6  (需要外部理论映射)
  通过对角线法发现:    depth = 0.8  (需要构造反例，更深)
  通过不动点检测发现:  depth = 1.0  (需要元层次分析，最深)

若同一点被多种方法发现，取最大 depth。
```

**Colony-021 基线:** 5 个盲点 (B1-B5)，全部由外部触媒法发现 → D_blind = 0.6

#### Q_axiom — 公理生成质量

```
Q_axiom = (1/N_axiom) · Σₖ IBEₖ

IBEₖ = 信息广播效率，计算自 Colony-001 提案定义:
  IBE = (F_t中不可证但A使其可证的高价值命题数) / (A的信息复杂度)

实际操作化（因 IBE 本身难以精确计算）:
  IBE_proxy = w_theory·C_theory + w_type·S_type + w_compat·ICS
  
  其中:
    C_theory = min(1.0, 理论引用数 / 3)     — 至少3个跨领域理论依据
    S_type    = 公理类型评分:
                  Type D (观察者重构):  1.00
                  Type A (维度扩展):    0.85
                  Type C (系统边界):    0.70
    ICS       = core_self 兼容性         — 通常 ∈ [0.95, 1.0]
    
    w_theory=0.4, w_type=0.35, w_compat=0.25
```

**Colony-021 基线:**
- AX-021-001: IBE_proxy = 0.4·1.0 + 0.35·1.00 + 0.25·0.98 = 0.995 → 实际 IBE=0.87
- AX-021-002: IBE_proxy = 0.4·1.0 + 0.35·0.85 + 0.25·0.96 = 0.938 → 实际 IBE=0.72
- AX-021-003: IBE_proxy = 0.4·1.0 + 0.35·0.70 + 0.25·0.97 = 0.888 → 实际 IBE=0.65
- Q_axiom = (0.87 + 0.72 + 0.65) / 3 = 0.747

#### E_dim — 维度扩展比

```
E_dim = (|DS_new| - |DS_old|) / |DS_old|

其中:
  |DS_old| = 跳前评估维度数
  |DS_new| = 跳后评估维度数（含二阶维度）

若跳后维度变化率本身成为新维度（AX-021-002），
各生成一个 D^(2) 算为 +0.5 维（因为它是衍生的，不是全新的）。
全新功能维度（如 Colony 交叉评估）算为 +1.0 维。
```

**Colony-021 基线:**
- |DS_old| = 5 (L5_SHI, MEM_COMP, SYNC_RATE, PRED_ACC, EXEC_GAP)
- AX-021-001 增加: +1.0 功能维 (探测协议能力)
- AX-021-002 增加: +2.5 二阶维 (5 维各 0.5，后续按需激活)
- AX-021-003 增加: +1.0 功能维 (Colony 交叉进化能力)
- |DS_new| = 5 + 1.0 + 2.5 + 1.0 = 9.5
- E_dim = (9.5 - 5) / 5 = 0.90

### 1.5 M_G 综合计算

**Colony-021 基线:**
```
M_G = 0.25·0.833 + 0.20·0.600 + 0.30·0.747 + 0.25·tanh(0.90)
    = 0.208 + 0.120 + 0.224 + 0.25·0.716
    = 0.208 + 0.120 + 0.224 + 0.179
    = 0.731
```

**解读:** M_G ∈ [0, 1]，0.731 属于"重大跳跃"区间。历史上第一次自主哥德尔跳。

### 1.6 M_G 分级参考

| M_G 区间 | 等级 | 含义 |
|:--------:|:----:|------|
| [0, 0.15) | 微跳 | 发现微小盲点，生成辅助性公理 |
| [0.15, 0.35) | 小跳 | 发现结构性盲点，1条高质量公理 |
| [0.35, 0.60) | 中跳 | 2-3条公理，涉及维度扩展 |
| [0.60, 0.85) | **大跳** | 多公理+维度扩展+GS重症解决 |
| [0.85, 1.00] | 巨跳 | 范式级变革，系统核心重构 |

## 2. 盲点发现率 BDR (Blindspot Discovery Rate)

### 2.1 定义

衡量哥德尔引擎搜索盲点的效率——在多大的搜索空间中找到了多少可转化为公理的有效盲点。

### 2.2 子指标

| 子指标 | 符号 | 定义 | 取值范围 |
|--------|------|------|:--------:|
| 搜索方法覆盖率 | C_method | 使用了 GE 三种搜索方法的比例 | [0, 1] |
| 候选生成密度 | D_candidate | 每种方法平均发现盲点数 | [0, ∞) |
| 盲点转化率 | R_convert | 盲点转化为公理的比例 | [0, 1] |
| 理论扫描覆盖率 | C_theory | 已扫描理论占可用理论库的比例 | [0, 1] |

### 2.3 计算公理

```
BDR = C_method · D_candidate_norm · R_convert · C_theory

其中:
  C_method = N_methods_used / 3     (3 = 外部触媒, 对角线, 不动点)
  
  D_candidate_norm = tanh(N_blindspots / N_methods_used / 3)
                    (归一化，期望每种方法约3个候选时为tanh(1)≈0.762)
  
  R_convert = N_axioms / N_blindspots
  
  C_theory = N_theories_scanned / N_theories_available
```

### 2.4 Colony-021 基线

```
C_method   = 3/3 = 1.0                    (三种方法全部使用)
D_candidate_norm = tanh(5/3/3) = tanh(0.556) = 0.506
R_convert  = 3/5 = 0.6                    (5 盲点 → 3 公理)
C_theory   = 5/5 = 1.0                    (全部 5 个理论已扫描)

BDR = 1.0 · 0.506 · 0.6 · 1.0 = 0.304
```

### 2.5 BDR 分级参考

| BDR 区间 | 等级 | 含义 |
|:--------:|:----:|------|
| [0, 0.10) | 低效 | 大量盲点未转化，或搜索不充分 |
| [0.10, 0.30) | 正常 | 搜索-转化链工作正常 |
| [0.30, 0.50) | **高效** | 盲点精准定位并高效转化 |
| [0.50, 1.00] | 超高 | 可能搜索空间太小或转化过于激进 |

> 注: BDR 过高(>0.7)可能意味着搜索空间不足——系统太"容易"发现盲点，暗示维度本身就很少。

## 3. 公理存活率 ASR (Axiom Survival Rate)

### 3.1 定义

衡量生成公理从提案到持续运行的完整生命周期存活情况。一条公理被生成只是开始，它必须被采纳为规则、通过稳定期、并在后续世代中产生可测量的影响。

### 3.2 生命周期阶段

```
生成 (Generated) → 采纳 (Adopted) → 操作化 (Operationalized) → 存活 (Survived) → 繁荣 (Thriving)

  G: 哥德尔引擎输出公理
  A: 通过安全门禁，转为 MR 规则
  O: 规则首次触发并产生可观测效果
  S: 经过 N 代稳定期后仍活跃（未被降权/禁用）
  T: N 代后产生持续的正面 ESV 贡献
```

### 3.3 计算公理

```
ASR = R_adopt · (1 + β · R_sustain) · (1 + γ · R_thrive)

其中:
  R_adopt  = N_adopted / N_generated    (采纳率)
  R_sustain = N_survived / N_adopted     (存活率，在稳定期后)
  R_thrive  = N_thriving / N_survived    (繁荣率，持续贡献)
  
  β = 0.5   (存活权重折扣——存活比采纳难)
  γ = 0.3   (繁荣权重折扣——繁荣比存活更难)
```

### 3.4 时变 ASR

ASR 不是常数——一条公理可能在 gen+N 时存活但在 gen+2N 时死亡。

```
ASR(t) = ASR(t₀) · e^(-λt)

其中 λ = -ln(R_sustain) / N_stable  (衰减常数)
     N_stable = 30 (提案要求的公理稳定期)
```

### 3.5 Colony-021 基线（跳后即时，gen-115）

```
N_generated = 3   (AX-021-001, AX-021-002, AX-021-003)
N_adopted   = 3   (MR-023, MR-024, MR-025, 全部通过安全门禁)
N_survived  = 3   (gen-117 确认三条 MR 全部激活)
N_thriving  = ?   (gen-118 尚未到稳定期——需要 gen-145 才满 30 代)

R_adopt    = 3/3 = 1.0
R_sustain   = 3/3 = 1.0  (即时——稳定期验证中)
R_thrive    = N/A (待 gen-145 评估)

ASR_instant = 1.0 · (1 + 0.5 · 1.0) = 1.5  (初期全部存活)

预计 ASR 在 30 代后:
  若 3/3 仍存活且全部繁荣: ASR = 1.0 · (1+0.5·1.0) · (1+0.3·1.0) = 1.95
  若 2/3 仍存活且繁荣:     ASR = 1.0 · (1+0.5·0.667) · (1+0.3·1.0) = 1.73
  若 1/3 仍存活:           ASR = 1.0 · (1+0.5·0.333) · (1+0.3·1.0) = 1.52
```

### 3.6 ASR 分级参考

| ASR 区间 | 等级 | 含义 |
|:--------:|:----:|------|
| [0, 0.5) | 低存活 | 公理质量存疑，大量被废弃 |
| [0.5, 1.0) | 正常 | 部分存活，符合预期 |
| [1.0, 1.5) | 高存活 | 多数公理持续运作 |
| [1.5, 2.0] | **超高** | 全部公理存活且产生正面影响 |

## 4. 维度扩展效率 DEE (Dimension Expansion Efficiency)

### 4.1 定义

衡量哥德尔跳在评估空间维度扩展方面的效率——新维度是否"物有所值"，每个新增维度携带多少有效信息增益。

### 4.2 子指标

| 子指标 | 符号 | 定义 | 取值范围 |
|--------|------|------|:--------:|
| 扩展比 | R_exp | 新维度数 / 旧维度数 | [0, ∞) |
| 饱和压力 | P_sat | 触及天花板(≥0.95·max)的维度占比 | [0, 1] |
| 单位公理扩展效率 | E_per_ax | 每条公理贡献的新维度数 | [0, ∞) |
| 新维度信息密度 | Q_dim | 新维度的平均信息广播效率 | [0, 1] |

### 4.3 计算公理

```
DEE = R_exp_norm · Q_dim · (1 + P_sat) · E_per_ax_norm

其中:
  R_exp_norm = tanh(R_exp)
              (当新增维度与原始维度相当时 ≈0.762)
  
  P_sat = |{d ∈ DS_old: d ≥ 0.95·max_d}| / |DS_old|
         (天花板维度的占比，反映扩展需求的紧迫性)
  
  Q_dim = Σ IBE(dimension_expansion_axiom) / N_expansion_axioms
         (导致维度扩展的公理的 IBE 均值)
  
  E_per_ax_norm = tanh(E_per_ax / 3)
                 (期望每条扩展公理带来约3个新维度为高效)
```

### 4.4 Colony-021 基线

```
R_exp = 4.5 / 5 = 0.90
R_exp_norm = tanh(0.90) = 0.716

P_sat = 2/5 = 0.40  (L5_SHI=3.00, MEM_COMP=3.00 触及天花板)

全部三条公理均有维度扩展贡献 (dimension_impact > 0):
  AX-021-001 (IBE=0.87): 探测协议功能维 (+1.0)
  AX-021-002 (IBE=0.72): 二阶评估维度 (+2.5)
  AX-021-003 (IBE=0.65): Colony 交叉进化功能维 (+1.0)

Q_dim = (0.87 + 0.72 + 0.65) / 3 = 0.747

E_per_ax = 4.5 / 3 = 1.5
E_per_ax_norm = tanh(1.5/3) = tanh(0.5) = 0.462

DEE = 0.716 · 0.747 · (1 + 0.40) · 0.462
    = 0.716 · 0.747 · 1.40 · 0.462
    = 0.346
```

### 4.5 DEE 分级参考

| DEE 区间 | 等级 | 含义 |
|:--------:|:----:|------|
| [0, 0.10) | 低效 | 维度过剩或扩展质量差 |
| [0.10, 0.25) | 正常 | 维度扩展物有所值 |
| [0.25, 0.50) | **高效** | 每新维度携带丰富信息 |
| [0.50, 1.00] | 超高 | 维度扩展高度精准，但需警惕是否维度本就不足 |

## 5. 综合指标: 哥德尔跳质量指数 GLQ

### 5.1 定义

GLQ (Godel Leap Quality Index) 将四项指标融合为单一可比较数值，用于跨跳比较和进化趋势追踪。

### 5.2 计算公理

```
GLQ = w₁·M_G + w₂·BDR_norm + w₃·ASR_norm + w₄·DEE

其中归一化:
  BDR_norm = tanh(BDR / 0.3)   (以 0.3 为参考点，tanh(1)=0.762)
  ASR_norm = tanh(ASR / 2)     (以 2.0 为参考点)
  M_G 和 DEE 已在 [0, 1] 范围

权重 (校准自 Colony-021):
  w₁ = 0.30  — 跳跃幅度是整体变革强度的核心度量
  w₂ = 0.20  — 盲点发现反映搜索效率
  w₃ = 0.25  — 公理存活反映长期价值
  w₄ = 0.25  — 维度扩展反映结构性进步
```

### 5.3 Colony-021 基线 GLQ

```
M_G      = 0.702
BDR_norm = tanh(0.304/0.3) = tanh(1.013) = 0.766
ASR_norm = tanh(1.50/2.0)  = tanh(0.750) = 0.635
DEE      = 0.346

GLQ = 0.30·0.731 + 0.20·0.766 + 0.25·0.635 + 0.25·0.346
    = 0.219 + 0.153 + 0.159 + 0.087
    = 0.618
```

### 5.4 GLQ 分级参考

| GLQ 区间 | 等级 | 含义 |
|:--------:|:----:|------|
| [0, 0.20) | 微跳 | 系统微调，非真正哥德尔跳 |
| [0.20, 0.40) | 小跳 | 有意义的突破，但范围有限 |
| [0.40, 0.60) | 中跳 | 结构性变革，多公理多维度 |
| [0.60, 0.80) | **大跳** | 范式级变革，Colony-021 级别 |
| [0.80, 1.00] | 巨跳 | 系统核心重建，极罕见 |

## 7. 与现有系统集成

### 7.1 替代 GODEL_LEAP 二值

```
旧: GODEL_LEAP ∈ {0, 1}
新: GODEL_LEAP ∈ {
      "count": N,              // 累计跳跃次数 (替代旧值)
      "last_magnitude": M_G,   // 最近一次跳跃幅度
      "last_glq": GLQ,         // 最近一次综合质量
      "cumulative_glq": ΣGLQ,  // 累计质量 (终身学习度量)
      "trend": "ASCENDING"|"STABLE"|"DESCENDING"
    }
```

### 7.2 写入自评文件

每次 gen 自评时，GODEL_LEAP 维度不再是 0/1，而是:

```
| GODEL_LEAP | {"count":1, "M_G":0.702, "GLQ":0.615, "trend":"ASCENDING"} |
```

若该代未发生哥德尔跳，保持上次数值不变。

### 7.3 触发 GS-002 检测增强

原 GS-002 只看 5 维 Δ 方差。新增强:
- 如果 GLQ 连续 3 跳递减 → 触发"哥德尔跳退化"警报
- 如果 M_G 连续下降 → 提示盲点搜索空间可能在收缩
- 如果 ASR < 0.5 → 提示公理质量控制需要增强

## 8. 局限性

1. **IBE 精确计算困难:** 信息广播效率的原始定义涉及"F_t 中不可证但 A 使其可证的高价值命题数"，这在自动化系统中难以精确计数。当前使用 IBE_proxy 作为替代。

2. **维度计数主观性:** "功能维度"和"二阶维度"的 1.0 vs 0.5 区分依赖人工判断。

3. **ASR 需要时间:** 公理存活率在跳后即时只能给出 R_adopt，真正的 R_sustain 需要 30 代后评估。即时 ASR 是预测值。

4. **权重校准:** 当前权重基于单一样本 (Colony-021) 校准。随着更多哥德尔跳积累，权重应通过回归重新校准。
