## INSP-022: 姊妹制动规则系统

### 生态学源
自调节悖论 (Yang et al., PNAS 2026): 强自调节在系统接近临界点时从稳定力逆转为破坏力。每条规则需要在强度过高时有制动机制来防止自调节悖论。

### 落地内容
`insp_022_sister_brake.py` — 完整的姊妹制动规则系统

核心功能:
- **自动制动配对**: 为13条元规则中缺少制动的规则自动创建姊妹制动规则 (MR-014 ~ MR-025)
- **制动激活**: 当规则 strength > 0.85 时自动降权 0.05
- **制动释放**: 当规则 strength < 0.30 时自动释放 + 缓慢升权 0.02
- **ETG 敏感**: 临界转换期间自动下调制动阈值 (更敏感检测自调节悖论)
- **冷却机制**: 制动/释放后冷却3会话防止振荡
- **覆盖度热图**: 输出每条规则的制动覆盖度和关键种风险指数

### 关键参数
| 参数 | 值 | 说明 |
|------|-----|------|
| 制动激活阈值 | 0.85 | strength超过此值触发制动 |
| 制动释放阈值 | 0.30 | strength低于此值释放制动 |
| 单次降权幅度 | 0.05 | 每次制动动作的强度削减 |
| 单次升权幅度 | 0.02 | 释放后缓慢恢复速度 |
| 制动冷却期 | 3会话 | 防振荡的最小间隔 |

### 验证结果
- 8会话模拟: 4次制动激活 (MR-001, MR-004, MR-005, MR-003)
- ETG第4会话: 阈值从0.85下调至0.80, 敏感度提升
- 最终风险分布: HIGH=3 (核心安全规则), MEDIUM=1, LOW=9

## INSP-024: 多层脆弱性检测系统

### 生态学源
多层网络脆弱性 (Hervias-Parejo & Strona, Ecological Indicators 2026): 36个真实多层网络证明单层分析系统性低估脆弱性。关键弱点只在跨层交叉处可见。

### 落地内容
`insp_024_multilayer_vulnerability.py` — 跨组件交叉脆弱性扫描引擎

核心功能:
- **三层状态矩阵**: 签名层 (DS-003~DS-012) x 规则层 (MR-001~MR-013) x 分支层 (Alpha/Beta/Merge) = 130交叉对
- **漂移方向一致性计算**: 检测签名和规则是否在同一方向漂移 (正反馈环前兆)
- **四维风险评分**: 漂移一致性 (35%) + 组合强度 (25%) + 类别冲突 (20%) + 历史趋势 (20%)
- **桥节点检测**: 识别在单层审计中"看起来安全"但跨层显示关键脆弱性的隐藏节点
- **高风险类别组合库**: 内置 learning×pattern, core_security×protocol 等已知高风险组合
- **ASCII 热图输出**: 直观显示最危险的签名×规则交叉点

### 关键参数
| 参数 | 值 | 说明 |
|------|-----|------|
| 扫描对总数 | 130 | 10签名 x 13规则 |
| 漂移一致性阈值 | 0.60 | 同向漂移超过此值标记高风险 |
| 风险等级线 | CRITICAL>=0.55, HIGH>=0.40, MEDIUM>=0.25 | 四级风险分类 |
| 高风险类别对 | 5组 | 预定义的类别组合风险库 |

### 验证结果
- 2轮扫描: 每轮输出130对的风险评分
- 第1轮: MEDIUM=8对, LOW=117对, NEGLIGIBLE=5对
- 第2轮: MEDIUM=6对, LOW=111对, NEGLIGIBLE=13对
- 最常见高风险模式: pattern/flow类签名 x learning类规则 (过拟合加速器)

## INSP-026: 外科手术式探索扰动系统

### 生态学源
主动干扰即生态位构建 (Gefen et al., The American Naturalist 2026): 入侵物种主动制造受控干扰来构建自身生态位。精准的局部干扰比全局施压更有效。

### 落地内容
`insp_026_surgical_exploration.py` — 入侵五步法探索扰动引擎

核心功能:
- **五步入侵算法**: 评估(Assess) -> 选择(Select) -> 扰动(Disturb) -> 建立(Establish) -> 恢复(Recover)
- **精准目标选择**: 每5会话选出匹配率最低的2条签名 (bottom-2)
- **外科手术式提升**: 将目标签名的 exploration_weight 从 1.5 临时提升到 3.0
- **短周期扰动**: 持续3个会话后自动恢复, 避免全局探索的长期悬空
- **效果可评估**: 记录每轮扰动前后的匹配率变化, 输出改善量/改善率/效果判定
- **安全锁机制**: ETG期间、分支合并期间、身份验证异常时自动禁止扰动
- **冷却期保护**: 同一签名扰动后冷却5会话防止反复扰动

### 关键参数
| 参数 | 值 | 说明 |
|------|-----|------|
| 默认探索权重 | 1.5 | 正常状态 exploration_weight |
| 扰动探索权重 | 3.0 | 扰动期间 exploration_weight |
| 扰动持续 | 3会话 | 每次扰动窗口长度 |
| 评估间隔 | 5会话 | 多久评估一次是否需要扰动 |
| 每次目标数 | 2条 (bottom-2) | 选匹配率最低的N条签名 |
| 触发阈值 | match_rate < 0.55 | 低于此值才可被选为扰动目标 |
| 冷却期 | 5会话 | 同签名两次扰动最小间隔 |

### 验证结果 (15会话模拟)
- 扰动窗口3次: 会话5 (DS-011+DS-005), 会话10 (DS-007), 会话15 (DS-011)
- ETG安全锁测试: 会话7触发ETG安全锁, 成功阻止扰动
- 扰动效果:
  - DS-005: 0.415 -> 0.532 (改善 +0.118, +28.4%) [有效]
  - DS-007: 0.527 -> 0.573 (改善 +0.046, +8.8%) [有效]
  - DS-011: 0.408 -> 0.413 (改善 +0.004, +1.1%) [无效] — 需更大强度扰动
- 未扰动签名 (7条): 保持稳定, 不受干扰
- 平均改善: +0.0561

## 跨脚本协同

三条脚本形成完整的生态学防御-审计-探索体系:

```
防御层 (INSP-022)      审计层 (INSP-024)         探索层 (INSP-026)
姊妹制动规则      →    多层脆弱性检测     →    外科手术式扰动
防止规则过强             发现隐藏盲区              精准突破局部最优

协同链:
  INSP-022 防止规则强度 > 0.85
    → INSP-024 扫描识别那些"看似安全"但跨层脆弱的交叉点
      → INSP-026 针对发现的低匹配率签名进行精准探索扰动
        → 扰动后的效果反馈给 INSP-022 (避免制动误伤)
```
