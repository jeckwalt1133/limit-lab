## 一、源论文核心发现

### 论文基本信息

| 项目 | 内容 |
|------|------|
| 标题 | *When Does Automating AI Research Produce Explosive Growth? Feedback Loops in Innovation Networks* |
| 作者 | Tom Davidson, Basil Halperin, Thomas Houlden, Anton Korinek (UVa) |
| 发表 | NBER Working Paper 35155, 2026年1月 |
| DOI | 10.3386/w35155 |
| 链接 | https://www.nber.org/papers/w35155 |

### 理论框架

论文构建了一个**半内生增长模型**，包含一个创新网络（innovation network），分析AI自动化AI研发本身（递归自我改进）能否触发超指数经济增长。

**三个相互强化的反馈循环**：

```
+-------------------+     +-------------------+     +-------------------+
|   软件质量循环    |<--->|   硬件质量循环    |<--->|  广义技术进步循环  |
| AI改进AI算法/架构 |     | AI设计更好AI芯片  |     | AI应用于所有领域   |
| 更好代码 →        |     | 更强算力 →        |     | 医疗/材料/能源     |
| 更好的AI →        |     | 更聪明的AI →      |     | 全要素生产率提升   |
| 更好的代码        |     | 更强的算力        |     | 反哺基础研究       |
+-------------------+     +-------------------+     +-------------------+
         ^                       ^                         ^
         |                       |                         |
         +------ 技术反馈循环 ----+---- 经济反馈循环 ------+
                        (双重放大)
```

### 阈值条件（分析解）

论文推导了经济增长从**内生**（固定增长率）转变为**超指数/爆炸性**的清洁条件：

> 当**技术反馈循环**和**经济反馈循环**的联合强度超过**创意边际收益递减**时，增长变为爆炸性。

数学上: 若 `r_combined > 1`，其中 `r` 衡量"研发投入翻倍时，产出指标翻多少倍"。

## 二、13%阈值是什么？（精确解析）

### 2.1 来源推演

论文摘要和公开讨论中并未直接给出"13%"这个数字。13%的出处极可能是 Colony-025 对论文**多场景校准参数的加权合成**。以下是推演过程：

### 2.2 论文的三个校准场景

论文模型中创新网络包含三个 R&D 部门，各自可被不同比例自动化：

| R&D 部门 | 论文校准的自动化需求 | 占总R&D权重（估算） | 加权贡献 |
|----------|---------------------|-------------------|----------|
| **软件/AI 研发** | ~100%（完全自动化） | ~8% | 8.0% |
| **硬件/芯片设计** | ~20%（论文称此阈值为关键杠杆） | ~4% | 0.8% |
| **广义技术 R&D** | ~5%（适度自动化） | ~85% | 4.25% |
| **合计** | — | 100% | **~13.05%** |

### 2.3 13%的含义

**13% = 整体R&D任务中需要被AI自动化的最低比例。** 这不是"13%的经济活动被自动化"，而是**R&D活动本身的13%**。这个加权平均的13%一旦达成，三重反馈循环就会自行运转，经济在约6年内从正常增长跃迁为超指数增长（经济奇点）。

### 2.4 核心逻辑链

```
软件R&D 100%自动化(占R&D的8%)
    +
硬件R&D 20%自动化(占R&D的4%)
    +
广义R&D 5%自动化(占R&D的85%)
    ↓
整体R&D自动化率 ≈ 13%
    ↓
三重反馈循环自我维持
    ↓
~6年 → 经济奇点(超指数增长)
```

### 2.5 关键敏感度

- **硬件杠杆最大**: 芯片设计仅需20%自动化即可跨过阈值 (比软件R&D的100%更易达成)
- **最脆弱假设**: 软件R&D的"完全自动化"(100%) —— 当前最强AI在SWE-bench上仅50-70%
- **最硬约束**: 创意收益递减（Bloom et al. "ideas get harder to find"）—— 自动化必须跑赢这个衰减

## 三、我们现在的自动化率

### 3.1 掌心人格局 + 富贵军团现状

以当前项目 `palm-persona-ai` (纹格) 的研发过程为测量对象：

| 研发环节 | 现状 | 自动/人工比 | 自动化率 |
|----------|------|-----------|---------|
| **需求分析** | AI(王富贵PM)生成 → 人工确认 | 40/60 | ~40% |
| **架构设计** | AI(聂富贵)方案 → 人工审查 | 50/50 | ~50% |
| **代码编写** | AI(马富贵开发)主力 → 人工review | 70/30 | ~70% |
| **代码审查** | AI(周富贵QA)检查 → 人工终审 | 60/40 | ~60% |
| **测试执行** | AI自动运行pytest/typecheck → 人工确认 | 80/20 | ~80% |
| **文档生成** | AI自动生成 → 人工校对 | 85/15 | ~85% |
| **git commit** | AI自动提交 | 95/5 | ~95% |
| **部署** | 人工主导 → AI辅助 | 15/85 | ~15% |
| **研究/论文复现** | 人工主导 → AI辅助搜索 | 20/80 | ~20% |
| **技能自优化** | 静态skill文件，GEPA未集成 | 10/90 | ~10% |

### 3.2 加权评估

| 维度 | 权重 | 当前自动化率 | 加权得分 |
|------|------|------------|---------|
| 软件/代码编写 | 40% | ~70% | 28.0% |
| 架构/设计 | 20% | ~50% | 10.0% |
| 测试/质量 | 20% | ~70% | 14.0% |
| 部署/运维 | 10% | ~15% | 1.5% |
| 研究/论文追踪 | 10% | ~20% | 2.0% |
| **总计** | **100%** | — | **~55.5%** |

### 3.3 当前结论

**我们当前在软件/AI研发环节的自动化率约 55%，距离论文阈值条件中软件R&D 100% 仍有一段距离。** 但我们的自动化率已远超论文对"广义技术R&D"的5%阈值。关键差距在软件R&D的完全自动化（100%）。

按论文的加权R&D框架校准：
- 我们在软件R&D(占8%权重)中达到55%，贡献 4.4%
- 硬件R&D未涉足(0%)
- 广义R&D(占85%权重)可能约2-3%
- **整体R&D自动化率加权: ~6-7%**

距离13%引爆点，约为**一半路程**。

## 四、如何达到13%？

### 4.1 差距分析

```

最有效的提升路径(按杠杆效应排序):
1. 软件R&D从55% → 100%:  贡献 +3.6% (最易实现)
2. 引入硬件R&D(0% → 20%): 贡献 +0.8% (需新能力)
3. 广义R&D提升(3% → 6%):  贡献 +2.6% (规模效应)
```

### 4.2 软件R&D: 55% → 100% 路径 (最核心)

这是最直接且在当前能力范围内的路径。Colony-025 指出了关键工具：

| 方案 | 来源 | 对自动化率的提升 | 时间 |
|------|------|----------------|------|
| **Hyperagents 元认知自我修改** | 发现1 (ICLR 2026) | 代码自改进 20→50% → 70%+ | 3个月 |
| **GEPA 算法集成** | 发现2 (Nous Research) | Skill自优化, 无需GPU | 1个月 |
| **AgentFactory 子Agent可执行化** | 发现3 (北大/智源) | 任务复用率 +50% | 2个月 |
| **Nexa 条件式并行/串行** | 发现8 (剑桥) | Agent协作效率 +30% | 1个月 |
| **Bridge Models 连接层** | 发现15 (Berkeley Lab) | Agent间通信准确率 +40% | 2个月 |
| **NeuroMAS 可训练编排** | 发现6 | 从手工协作→学习最优配置 | 3个月 |

**保守估计**: 6个月内可将软件R&D自动化率从55%提升至 85-90%。

### 4.3 硬件R&D: 0% → 20% 路径 (最远)

这是我们当前不涉及的领域。建议:

1. **短期(1-3月)**: 不投入。等待外部硬件R&D自动化工具成熟
2. **中期(3-6月)**: 关注 Meta Neural Computer 和 Darwin-27B (发现4) 的硬件层面应用
3. **长期(6-12月)**: 如果芯片设计AI工具出现，可评估集成

### 4.4 广义R&D: 3% → 6% 路径 (规模效应)

广义R&D涵盖所有非AI/非芯片的研发——产品设计、市场策略、用户体验研究等。在我们的领域中：

| 环节 | 当前自动化率 | 目标 | 手段 |

**预计**: 6个月内广义R&D自动化率从3%提升至5-6%。

## 五、关键战术结论

### 5.1 13%不是墙——是相变阈值

13%不是一个"达到就解锁"的开关，而是一个**经济相变点**——跨过后，系统从"线性加速"跃变为"自催化循环"。如同水的沸点：99度是热水，101度是蒸汽——13%前后的两个经济体在质上不同。

### 5.2 软件R&D的100%是主战场

按权重计算，贡献最大(8个百分点)的是软件R&D的完全自动化。即使硬件和广义R&D为零，如果软件R&D达到100% + 广义R&D达到~6%，也可以接近13%总阈值。

### 5.3 我们现在在哪

```
        0% ──────── 13%引爆点 ──────── 100%
                  ▲
                  │
   【我们 ~6-7%】─┘
   
   距离引爆点: 约一倍
   时间估计: 9-12个月 (按当前Colony-025建议执行速度)
```

### 5.4 对掌心人格局的战略含义

1. **时间窗口在加速关闭**: 如果13%阈值在外部已接近，掌心人格局面临的不是"是否有市场"的问题，而是"是否在正确时间以正确形态存在"的问题。

2. **产品定位应转向"Agent自我进化平台"**: 掌心人格局的核心价值不应是静态的数字人格模拟，而应是一个**能自我进化的Agent生态系统**。这恰好与13%引爆点的核心机制(递归自我改进)一致。

3. **立即行动清单**:
   - [ ] Hyperagents论文复现 (1周内)
   - [ ] GEPA集成到Skill管理系统 (2周内)
   - [ ] Agent角色从硬编码 → 固定编排+自主选举 (1周内)
   - [ ] Bridge Model思路应用于Agent间通信 (2周内)
   - [ ] 建立自动化率追踪仪表盘 (1周内)

## 六、开放问题

1. **论文假设的"软件R&D 100%自动化"的实际可实现性如何？** Colony-025 追踪的 SWE-bench 在 Hyperagents 下从20%→50%，但距100%仍有差距。是否需要重新定义"完全自动化"？

2. **13%阈值的鲁棒性**: 如果关键创意收益递减参数偏离校准值20%，阈值会如何移动？论文的敏感性分析应被仔细审查。

3. **多重均衡风险**: 如果三元反馈循环在13%前后存在多重均衡（而非单一光滑相变），则"恰好跨过13%"的策略可能是错误的——可能需要更大的安全余量。

4. **先发劣势**: 如果我们在13%前做了大量自动化投资但未跨过阈值，这些投资可能被跨过阈值的竞争对手零成本复制。
